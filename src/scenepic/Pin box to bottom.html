Got you this is a super common issue.
If 100vh gives you a tiny scroll bar (like 1 inch), its almost always caused by extra margin/padding somewhere in the document.

Here are the exact fixes in order of most likely least likely.

1. Remove default body margin (MOST COMMON FIX)

Browsers add margin: 8px to <body> by default.
That alone will create the tiny scroll.

Add this:

css
html, body {
  margin: 0;
  padding: 0;
  height: 100%;
}

This usually fixes the 1-inch scroll issue instantly.

2. Check if ANYTHING outside the box adds spacing

Common culprits:

  • a header

  • a nav bar

  • a top-level <div> adding padding

  • global * { margin: ... }

You can test by temporarily giving the <html> background a color:

css
html {
  background: red;
}

If you see a red strip above or below your main container, you found the cause.

3. Use min-height: 100vh instead of height

Sometimes height: 100vh is too strict and causes overflow by a pixel.

Try:

css
.main-box-container {
  min-height: 100vh;
  height: auto;
}

Removes scrollbar
Still fills the full viewport
Doesnt push overflow

4. Use overflow: hidden on the body

If the scroll is literally 12px and caused by rounding:

css
body {
  overflow: hidden;
}

But only use this if the MUD layout itself handles all scrolling internally.
